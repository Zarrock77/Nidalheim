# Client UE5 — Aperçu

Le client jeu Unreal Engine 5.6 (`Nidalheim.uproject`) vit dans un **depot Perforce séparé** du monorepo backend. Le depot s'appelle `main` et est hébergé sur le VPS Epitech (cf [Workflow Perforce](/docs/unreal-engine/perforce)). Côté dev local, le stream/workspace s'appelle `JJ_main` (chemin par défaut `C:\Users\jj\Perforce\JJ_main\`). Il consomme les services backend documentés dans le reste de cette doc :

- **`api-auth`** via REST + Device Authorization Grant pour l'auth joueur ([détails](/docs/unreal-engine/auth))
- **`api-game`** via WebSocket `/text` et `/audio` pour les PNJ pilotés par IA ([voice pipeline](/docs/backend/voice-pipeline))

## Source layout (`Source/Nidalheim/`)

Organisation **Lyra-style** par domaine fonctionnel — pas par classe UE.

```
Source/Nidalheim/
├── Nidalheim.h/cpp                  Module + log category (LogNidalheim)
├── Nidalheim.Build.cs               Dépendances modules + include paths
│
├── Auth/                            Device Authorization Grant client
│   ├── NidalheimAuthTypes.h         Enums + structs Blueprint
│   ├── NidalheimAuthClient.cpp      HTTP bas niveau
│   ├── NidalheimAuthStorage.cpp     Persistance tokens
│   └── NidalheimAuthSubsystem.cpp   UGameInstanceSubsystem — orchestrator
│
├── Character/
│   ├── NidalheimCharacterBase       Base GAS (ASC + AttributeSet + abilities)
│   ├── NidalheimPlayerCharacter     Capsule + character movement (input/caméra en BP)
│   ├── NidalheimNPCCharacter        PNJ "cerveau" — double WebSocket text+audio
│   ├── NidalheimInGameNPC           PNJ posé en niveau, InteractionSphere → ShowChat
│   └── NidalheimAudioCaptureComponent
│
├── Player/
│   └── NidalheimPlayerController    IMC mappings (AddMappingContext)
│
├── System/
│   └── NidalheimGameMode
│
├── AbilitySystem/                   GAS — convention Nidalheim*
│   ├── NidalheimAbilitySystemComponent
│   ├── Abilities/NidalheimGameplayAbility
│   └── Attributes/NidalheimAttributeSet
│
└── UI/
    ├── NidalheimChatSubsystem       ULocalPlayerSubsystem — propriétaire NPC + ChatWidget
    ├── NidalheimChatWidget          Slate (pas UMG)
    ├── NidalheimAuthUISubsystem     Affichage user_code, état d'auth
    └── NidalheimAuthWidget
```

> Le dossier `TP_ThirdPerson/` (template UE5) coexiste mais n'est **pas** la logique de prod. À supprimer dès que possible.

Détails sur l'organisation `Content/` et le naming dans [Conventions](/docs/unreal-engine/conventions).

## Build.cs — modules

**Public** : `Core`, `CoreUObject`, `Engine`, `InputCore`, `EnhancedInput`, `AIModule`, `StateTreeModule`, `GameplayStateTreeModule`, `GameplayTasks`, `GameplayTags`, `GameplayAbilities`, `UMG`, `Slate`, `Sockets`, `WebSockets`, `AudioMixer`, `AudioCapture`, `Niagara`, `NiagaraCore`, `ChaosNiagara`, `NetCore`.

**Privé** : `SSL`, `HTTP`, `SlateCore`, `Json`, `JsonUtilities`.

`PublicIncludePaths` inclut explicitement les sous-dossiers (`Auth/`, `Character/`, `AbilitySystem/`, `Player/`, `UI/`, `System/`) pour permettre les `#include "X.h"` sans préfixe de chemin.

## Configuration runtime

`Config/DefaultGame.ini`, section `[Nidalheim]` :

```ini
AuthApiUrl="https://api-auth.nidalheim.com"
WebVerificationUrl="https://www.nidalheim.com/link"
WebsocketTextBaseUrl="wss://api-game.nidalheim.com/text"
WebsocketAudioBaseUrl="wss://api-game.nidalheim.com/audio"
AudioCaptureVolume=0.1
```

Pour pointer vers un backend local (cf [Setup](/docs/backend/setup-project)) : remplacer par `http://localhost:3001` / `ws://localhost:3002/text` etc.

## Coordination cross-repo backend ↔ UE5

Toute modification de protocole doit être appliquée des **deux côtés** :

| Modification | Côté backend | Côté UE5 |
|---|---|---|
| Forme du JWT (claims) | `services/api-auth/src/services/jwt.ts` + `api-game/src/auth.js` | `FNidalheimAuthSubsystem` (lecture des claims) |
| Query params WebSocket | `services/api-game/src/server.js` | `BuildWsUrl` dans `NidalheimNPCCharacter.cpp` |
| Endpoints device flow | `services/api-auth/src/routes/device.ts` | `FNidalheimAuthClient` |
| Schema `npcs` | `services/db/src/schema/npcs.ts` + migration | `NpcId` UPROPERTY (juste un FString, pas de breaking) |
| `JWT_SECRET` rotation | Mettre à jour `infra/.env` sur les deux services | Force re-login (tous les access tokens existants invalidés) |
